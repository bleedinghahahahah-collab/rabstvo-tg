# Подполье — Telegram Mini App

Полноценная игра «Рабство» в виде Telegram Mini App: бот + сервер + база данных + фронтенд с вкладками, топом игроков и реферальной системой.

## Что внутри

```
rabstvo-tg/
├── src/
│   ├── server.js   — Express-сервер, все API-эндпоинты, раздача фронтенда
│   ├── bot.js      — Telegram-бот (/start, обработка реферальной ссылки)
│   ├── db.js       — простое хранилище данных в файле `data.json` (без баз данных и без компиляции чего-либо — работает "из коробки" на любой системе)
│   ├── game.js      — игровая экономика: доход, стоимость захвата, шанс успеха, восстания
│   └── auth.js      — проверка подлинности initData от Telegram (защита API)
├── public/
│   ├── index.html   — разметка мини-аппы (шапка-гроссбух + 5 вкладок + нижний таббар)
│   ├── style.css    — дизайн (тёмная тема «подпольной бухгалтерии»)
│   └── app.js       — вся клиентская логика и запросы к API
├── .env.example
└── package.json
```

## Игровая механика

- **Баланс и доход**: у каждого игрока копится доход в час, зависящий от базового дохода + количества людей, которые ему принадлежат.
- **Рынок**: список свободных игроков. Захват стоит монет (цена растёт от их уровня защиты и количества уже подвластных им людей) и имеет шанс успеха — тем выше, чем выше твой уровень защиты и число уже захваченных людей относительно цели.
- **Мои люди**: тех, кого ты захватил, можно отпустить бесплатно. Они сами могут выкупить свободу за монеты (эндпоинт `/api/ransom`), если это ты сам — в разделе «Профиль».
- **Защита**: прокачивается за монеты, растёт по экспоненте — снижает шанс, что тебя захватят, и шанс, что твои люди взбунтуются.
- **Восстание**: раз в 10 минут фоновый таймер (`runRebellionTick`) проверяет каждого захваченного игрока — с небольшим шансом (тем меньше, чем выше защита владельца) он сбегает на свободу сам.
- **Топ игроков**: рейтинг по балансу, показывает титул и число людей в подчинении.
- **Приглашения**: у каждого своя ссылка `t.me/BOTNAME?start=ref_<id>`. Перешедший по ней **сразу поступает в услужение** пригласившему — реферальная система напрямую встроена в механику захвата, а не оторвана от неё.
- **Дневной бонус**: раз в 20+ часов, с растущей наградой за серию дней подряд.
- **Титулы**: от «Бродяги» до «Императора подполья» — чисто косметические, зависят от числа подвластных людей.

## Что я бы добавил от себя (частично уже реализовано, частично — на будущее)

Уже в коде:
- Привязка приглашений к самой механике владения (а не просто бонус за клик по ссылке) — это сильно увеличивает конверсию инвайтов, потому что приглашающий сразу получает выгоду.
- Уведомления от бота в реальном времени: «тебя пытались захватить», «твой человек сбежал», «тебя отпустили» — это самое дешёвое и мощное средство возврата игроков в приложение (retention).
- Восстания — они создают постоянное лёгкое напряжение и причину регулярно прокачивать защиту, а не просто копить монеты.

Идеи на будущее (не реализовано, но легко добавить на этой архитектуре):
1. **Аукцион** — можно выставлять своих людей на торги, а не просто отпускать; в схеме БД уже есть заготовка таблицы `auctions`.
2. **Кланы/альянсы** — совместная казна и отдельный клановый топ, чтобы игра была не только 1-на-1.
3. **Сезоны** — раз в месяц обнуление баланса с сохранением титула/ачивки за прошлый сезон, чтобы новые игроки не упирались в старожилов.
4. **Более мягкие названия** — если решишь смягчить тематику под модерацию Telegram/App-сторов, термины «раб/поработить» можно заменить на «вассал/завербовать» без изменения механики — это чисто косметика в текстах `bot.js` и `app.js`.
5. **Push через бота с расписанием** — например, раз в день присылать «сводку гроссбуха»: сколько заработал, кто пытался тебя захватить.

## Установка и запуск

1. Установи зависимости:
   ```bash
   npm install
   ```
2. Создай бота через [@BotFather](https://t.me/BotFather) → `/newbot`, получи токен.
3. Скопируй `.env.example` в `.env` и заполни:
   ```
   BOT_TOKEN=твой_токен_от_BotFather
   WEBAPP_URL=https://твой-домен-или-туннель
   PORT=3000
   USE_WEBHOOK=false
   ```
   Telegram Mini Apps **обязательно требуют HTTPS** — для локальной разработки прокинь порт через [ngrok](https://ngrok.com) или [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/) и укажи выданный URL в `WEBAPP_URL`.
4. В @BotFather включи кнопку меню: `/mybots` → выбери бота → `Bot Settings` → `Menu Button` → укажи тот же `WEBAPP_URL`. Это позволит открывать игру не только из `/start`, но и из кнопки меню чата.
5. Запусти:
   ```bash
   npm start
   ```
6. Напиши боту `/start` в Telegram — появится кнопка «Открыть игру».

### Деплой в продакшн

- Подойдёт любой хостинг с Node.js и постоянным HTTPS-адресом: Railway, Render, Fly.io, VPS + nginx + certbot.
- Поставь `USE_WEBHOOK=true` и после старта один раз зарегистрируй вебхук (можно curl'ом):
  ```
  https://api.telegram.org/bot<ТОКЕН>/setWebhook?url=<WEBAPP_URL>/webhook/<ТОКЕН>
  ```
- `data.json` — файл со всеми данными игроков, лежит рядом с проектом. Для нагрузки выше нескольких тысяч активных игроков стоит перейти на настоящую базу данных, но для старта и проверки идеи простого JSON-файла более чем достаточно, а главное — не требует установки ничего дополнительного на компьютер.

## Безопасность

Каждый запрос к API проверяется через `verifyInitData` (`src/auth.js`) — это официальный способ Telegram подтвердить, что запрос действительно пришёл из открытого в Telegram мини-приложения, а не подделан извне. Без валидной подписи запрос отклоняется (401).
